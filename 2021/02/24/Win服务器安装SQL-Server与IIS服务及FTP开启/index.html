Win服务器安装SQL Server与IIS服务及FTP开启

SQL server服务卸载(各版本均可)

由于第一次安装失败,所以卸载后重装,步骤记录

  1. 运行> 输入services.msc,查看所有SQL server服务并停止

  2. 在控制面板中查看类别选择,进入程序卸载界面

  3. 在右上角输入sql server,卸载所有与SQL server相关的程序

  4. 删除SQL server相关注册表

    win+r输入regedit进入注册表

    1. HKEY_LOCAL_MACHINE\SYSTEM\ControlSet001\Control\Session Manager中删除PendingFileRenameOperations

    2. 删除HKEY_CURRENT_USER\SOFTWARE\Microsoft\下的 SQL server相关

    3. HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\下面所有关于SQL server的子项删除

    4. 删除HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\MSSQLServer

  5. 删除相关文件夹

    1. 删除安装SQL server的物理路径,包含默认安装路径下所有

  6. 卸载完成后重启电脑

安装IIS服务

由于安装包较大安装此服务,开启ftp传输

  1. 服务器管理器中点击添加角色

  2. 一直下一步到服务器角色,勾选Web 服务器(IIS)

  3. 在功能中勾选Net3.5及4.5及其所有子项

  4. 点击下一步后,会有勾选的,因为我已安装所以无图. 勾选时确保所有子项\展开项全部勾选

  5. 如果没有联网就需要下载Net3.5,并在指定备用源路径中指定路径

https://wwi.lanzous.com/b0108j5ti
密码:eog3

Net3.5压缩包

搭建FTP站点

在被传送的服务器中搭建ftp服务

  1. 运行> inetmgr进入IIS管理器,添加FTP站点

  2. 配置站点名与允许被访问的路径

  3. 选择绑定的ip,更改端口. 并选择无SSL

  4. 全勾选,选择所有用户,虽然不安全,但是我用完删掉

  5. 此时便可以在跳板机中通过xftp连接,传文件

安装Sqlserver

  1. 没报错就是点下一步
  2. 有全选就全选
  3. 实例可以直接选默认
  4. 选择服务时,有一个时禁用,可以改为手动
  5. 混合模式,自己设置sa密码
  6. 有添加当前用户直接点
  7. 控制器名称可以不写,直接下一步
