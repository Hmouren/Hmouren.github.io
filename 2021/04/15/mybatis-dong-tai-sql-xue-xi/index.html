动态 SQL 是 MyBatis 的强大特性之一。

  • if
  • choose (when, otherwise)
  • trim (where, set)
  • foreach
  • 等等

Mybatis动态SQL

IF

使用动态SQL最常见的情景是根据条件包含where子句的一部分，eg：

如果未传入color,price，则只按name查询。

<select id="findCommodity" resultType="Commodity">
	select * from Commodity where name="手机"
    <if test="color != null">
    	and color like #{color}
    </if>
    <if test="price != null">
    	and price like #{price}
    </if>
</select>

choose、when、otherwise

适用于只想从多个条件中选择一个使用。

当传入color，则按color查询

当传入price，则按price查询

如果都没有传入，则按recommend=1，进行查询。（与其返回随机商品，不如返回推荐商品）

<select id="findCommodity" resultType="Commodity">
	select * from Commodity where status="active"
    <choose>
    	<when test="color != null">
        	and color like #{color}
        </when>
        <when test="price != null">
        	and price like {price}
        </when>
        <otherwise>
        	and recommend = 1
        </otherwise>
    </choose>
</select>

trim、where、set

①where 元素只会在子元素返回任何内容的情况下才插入 “WHERE” 子句。而且，若子句的开头为 “AND” 或 “OR”，where 元素也会将它们去除(会自动去除第一个and 或者 or)。

②set 元素可以用于动态包含需要更新的列，忽略其它不更新的列。

③trim可用于自定义where 和 set

where

即时status为空，color！=color，where元素在生成sql语句时会自动将第一个and或者or去除！

<select id="findCommodity" resultType="Commodity">
	select * from Commodity
    <where>
    	<if test="status != null">
        	status like #{status}
        </if>
        <if test="color != null">
        	and color like #{color}
        </if>
        <if test="price != null">
        	and price like #{color}
        </if>
    </where>
</select>

set

<update id="updateCommodity">
	update Commodity
    <set>
    	<if test="status != null">status=#{status},</if>
        <if test="color != null">color=#{color},</if>
        <if test="price != null">price=#{price}</if>
    </set>
    where id=#{id}
</update>

trim

  1. 完成where功能

    prefix：定义前缀；

    prefixoverride：去掉第一个and或者or

    <select id="findCommodity" resultType="Commodity">
	select * from Commodity
    <trim prefix="WHERE" prefixoverride="AND |OR">
    	<if test="status != null">and status=#{status}</if>
        <if test="color != color">and color=#{color}</if>
        <if test="price != null">and price=#{price}</if>
    </trim>
</select>

  2. 完成or功能

    prefix：定义前缀

    suffixoverride：去除最后一个逗号

    suffix：定义后缀

    <update id="updateCommodity">
	<trim prefix="set" suffixoverride="," suffix="where id=#{id}">
    	<if test="status != null">status=#{status},</if>
        <if test="color != null">color=#{color},</if>
        <if test="price != null">price=#{price},</if>
    </trim>
</update>

foreach

参考链接

动态SQL的另一个常见使用场景就是对集合进行遍历（尤其是在构建IN条件语句的时候）

foreach元素的属性主要有：item、index、collection、open、separator、close、item。

  1. item：表示集合中每一个元素进行迭代时的别名，也就是常看的value值（本次迭代获取的元素）；

  2. index：指定一个名字，用于表示在迭代过程中，每次迭代到的位置。（迭代次数）；

  3. open：表示该语句以什么开始；

  4. separator：表示每次进行迭代间以什么符号作为分隔符；

  5. close：表示以什么结束；

  6. collection：该属性必须指定，且在不同情况下。该属性的值不同，具体如下：

collection属性不同情况区别：

  1. 如果传入的时单参数且参数类型时一个List时，collection属性值为list；

    //与之对应的mappper代码
public List<Commodity> dynamicForeachTest(List<Integer> ids);

  2. 如果传入的时单参数且参数类型是一个array数组的时候，collection的属性值为array；

    //与之对应的mappper代码
public List<Commodity> dynamicForeachTest(int[] ids);

  3. 如果传入的参数时多个的时候，我们就需要将它们封装成Map。单参数也可以封装成map。map的key时参数名。所以此时collection属性值就是传入的List或array对象在自己封装的map里面的key

    //与之对应的mappper代码
public List<Commodity> dynamicForeachTest(Map<String,Object> params);

简单demo

<select id="selectPostIn" resultType="domain.blog.Post">
  SELECT *
  FROM POST P
  WHERE ID in
  <foreach item="item" index="index" collection="list"
      open="(" separator="," close=")">
        #{item}
  </foreach>
</select>

script

通过script元素，可以在带注解的映射器接口类中使用动态sql

    @Update({
            "<script>",
            "update Commodity",
            "<set>",
            "<if test='status != null'>status=#{status},</if>",
            "<if test='color != null'>color=#{color},</if>",
            "<if test='price != null'>price=#{price}",
            "</set>",
            "where id=#{id}",
            "</script>"
    })
void updateCommodity(Commodity commodity)

bind

bind元素允许你在OGNL表达式以外创建一个变量，并将其绑定到当前的上下文中

<select id="findCommodity" resultType="Commodity">
	<bind name="title" value="%"+Commodity.getTitle()+"%" />
    select * from Commodity
    where title like #{title}
</select>

多数据库支持

如果配置了 databaseIdProvider，你就可以在动态代码中使用名为 “_databaseId” 的变量来为不同的数据库构建特定的语句。

<insert id="insert">
  <selectKey keyProperty="id" resultType="int" order="BEFORE">
    <if test="_databaseId == 'oracle'">
      select seq_users.nextval from dual
    </if>
    <if test="_databaseId == 'db2'">
      select nextval for seq_users from sysibm.sysdummy1"
    </if>
  </selectKey>
  insert into users values (#{id}, #{name})
</insert>
