Linux Centos7.5安装使用Elasticsearch和IK分词器

1.简介

官网

Elastic有一条完整的产品线及解决方案：Elasticsearch、Kibana、Logstash等，前面说的三个就是大家常说的ELK技术栈，（是最新的日志解决方案）

·Elasticsearch是Elastic Stack的核心

官网

Elasticsearch的特点

​ ·分布式，无需人工搭建集群（solr就需要人为配置，使用Zookeeper作为注册中心）；

​ ·Restful风格，一切API都遵循Rest原则，容易上手；

​ ·近实时搜索。数据更新在Elasticsearch中几乎是完全同步的；

2.安装Elasticsearch

·首先前提条件是，虚拟机安装有jdk且版本在1.8及以上

2.1新建一个用户leyou

·出于安全考虑，Elasticsearch默认不允许以root账户运行

创建用户leyou

useradd leyou

设置密码为leyou

passwd leyou

切换用户

su - leyou

2.2解压安装Elasticsearch

·在root中新增用户之后，会自动在/home/目录下创建对应用户名的文件夹

·将安装包放于/home/leyou/下

·退出到root用户中，修改文件的权限为leyou用户leyou组

chown leyou:leyou elasticsearch-6.2.4.tar.gz

·修改文件的操作权限

chmod 755 elasticsearch-6.2.4.tar.gz

·使用leyou用户，解压配置elasticsearch-6.2.4.tar.gz

su - leyou
tar -zxvf elasticsearch-6.2.4.tar.gz
mv elasticsearch-6.2.4.tar.gz elasticsearch

·修改配置文件

vim /elasticsearch/config/jvm.options
#修改如下
-Xms512m
-Xmx512m
vim /elasticsearch/config/elasticsearch.yml
#修改如下
path.data: /home/leyou/elasticsearch/data
path.logs: /home/leyou/elasticsearch/logs
network.host: 0.0.0.0

在elasticsearch中创建对应的文件夹

mkdir -p data

启动

./elasticsearch/bin/elasticsearch

2.3启动后报错解决

2.3.1文件权限不足

max file descriptors [4096] for elasticsearch process likely too low, increase to at least [65536]

登录root用户，修改如下

vim /etc/security/limits.conf
#修改如下
* soft nofile 65536
* hard nofile 131072
* soft nproc 4096
* hard nproc 4096

2.3.2线程数不够

max number of threads [1024] for user [leyou] is too low, increase to at least [4096]

修改如下

vim /etc/security/litmits.d/20-nproc.conf
#将
* soft nproc 1024
#改为
* soft nproc 4096

2.3.3进程虚拟内存

max virtual memory areas vm.max_map_count [65530] likely too low, increase to at least [262144]

修改如下

vm.max_mao_count:限制一个进程可以拥有的VMA（虚拟内存区域）数量

vim /etc/sysctl.conf
#添加如下
vm.max_map_count=655360

添加成功之后，执行sysctl -p

重启xshell终端

使用leyou用户开启服务

[leyou@localhost ~]$ ./elasticsearch/bin/elasticsearch

成功启动后会绑定两个端口

·9300：集群结点间通信接口

·9200：客户端访问接口

访问http://192.168.77.131:9200/

2.4还有一种错误，内核过低

如果使用的是centos7以下版本，egcentos6，其linux内核版本为2.6。而Elasticsearch的插件要求至少3.5以上版本，所以此时需要禁用这个插件

修改elasticsearch.yml文件，在最下面添加如下配置

bootstrap.system_call_filter:false

3.安装IK分词器

将安装包elasticsearch-analysis-ik-6.2.4.zip，上传至elasticsearch的plugins目录下，并且解压重命名为ik-analyzer

cd ./plugins/
[leyou@localhost plugins]$ unzip elasticsearch-analysis-ik-6.2.4.zip
mv elasticsearch ./ik-analyzer
rm -rf elasticsearch-analysis-ik-6.2.4.zip 
#重启elasticsearch即可
